# myfeeder UI Design Spec

## Overview

A desktop-first, three-panel web UI for myfeeder — a power-user feed reader built with React + TypeScript, served from the existing Spring Boot backend as a monorepo SPA.

## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Core experience | Power user / organizational | Boards, keyboard shortcuts, integrations front-and-center |
| Platform | Desktop-first web | Mobile deferred; React Native remains an option |
| Frontend framework | React + TypeScript | Complex stateful UI, largest ecosystem, React Native path open |
| Architecture | Monorepo SPA in Spring Boot | Single deployable JAR, Vite dev server proxies to backend |
| Layout | Three-panel (Feedly Classic) | Feed tree → article list → reading pane; maximum information density |
| Feed organization | Folders + boards | Folders group feeds; boards curate articles across feeds |
| Keyboard shortcuts | Full Vim-style | j/k navigation, g-chords for go-to, / for search |
| Article rendering | Embedded + open original | Sanitized HTML in-app, button/shortcut to open source URL |
| Integrations (v1) | Raindrop + generic share/export | Raindrop built out; share menu with copy link, export |

## Build Integration

- `vite.config.ts` proxies `/api/**` to `localhost:8080` during development
- A Gradle task (`npmBuild`) runs `npm run build` and copies output to `src/main/resources/static/`
- `./gradlew build` produces a single fat JAR with frontend included
- Spring Boot serves the SPA and falls back to `index.html` for client-side routing via a `WebMvcConfigurer`

**Dev workflow:**
- Terminal 1: `./gradlew bootTestRun` (backend with Testcontainers)
- Terminal 2: `cd src/main/frontend && npm run dev` (Vite on port 5173, proxying API calls)

## Data Flow & State Management

### Server State — TanStack Query

All data from the backend is managed by TanStack Query, providing caching, background refetching, optimistic updates, and request deduplication.

| API Client | TanStack Query Hook |
|---|---|
| `feedsApi.getAll()` | `useFeeds()` |
| `feedsApi.subscribe(url)` | `useSubscribeFeed()` |
| `articlesApi.list(params)` | `useArticles(feedId, filters)` |
| `articlesApi.updateState()` | `useUpdateArticleState()` |
| `articlesApi.markRead()` | `useMarkRead()` |
| `articlesApi.saveRaindrop()` | `useSaveToRaindrop()` |
| `foldersApi.list()` | `useFolders()` |
| `boardsApi.list()` | `useBoards()` |
| `integrationsApi.get()` | `useIntegrations()` |

### Client State — Zustand

UI state that doesn't exist on the server:

- `selectedFeedId` / `selectedFolderId` — active feed/folder in feed panel
- `selectedArticleId` — highlighted article in list
- `panelWidths` — resizable divider positions (persisted to localStorage)
- `expandedFolders` — open/closed state of folders in tree (persisted to localStorage)
- `keyboardFocus` — which panel currently has keyboard focus

### Data Flow Example

1. User clicks "Hacker News" in FeedPanel → sets `selectedFeedId` in Zustand
2. ArticleList renders `useArticles(selectedFeedId)` → TanStack Query fetches `GET /api/articles?feedId=3`
3. Query result is cached; subsequent clicks return instantly
4. User presses `j` → advances `selectedArticleId` in Zustand
5. ReadingPane renders the selected article's content
6. Article auto-marked as read → `useUpdateArticleState` fires `PATCH /api/articles/{id}` with `{read: true}`, optimistic update in query cache

### Optimistic Updates

Star, mark-read, and board-add operations update the UI immediately and reconcile with the server response. If the server call fails, the UI rolls back.

### Key Libraries

| Concern | Library | Rationale |
|---|---|---|
| Data fetching & cache | TanStack Query | Industry standard for React server state |
| Client state | Zustand | Lightweight, no boilerplate, works well with TanStack Query |
| HTML sanitization | DOMPurify | Secure-by-default XSS prevention for article content |
| Routing | React Router v7 | Client-side routing for feeds, boards, settings |
| Keyboard shortcuts | Custom hook | Simpler than a library for vim-style chords |

## Backend Changes

All additive — no modifications to existing code.

### New Entities

- **Folder** — `id`, `name`, `displayOrder`, `createdAt`
- **FeedFolder** — join mapping: `feedId`, `folderId` (a feed belongs to one folder, or none for "uncategorized")
- **Board** — `id`, `name`, `description`, `createdAt`
- **BoardArticle** — join mapping: `boardId`, `articleId`, `addedAt`

### New Endpoints

```
GET    /api/folders                        — list folders with feed counts
POST   /api/folders                        — create folder
PUT    /api/folders/{id}                   — rename folder
DELETE /api/folders/{id}                   — delete folder (feeds become uncategorized)
PUT    /api/feeds/{id}/folder              — move feed to folder

GET    /api/boards                         — list boards
POST   /api/boards                         — create board
PUT    /api/boards/{id}                    — update board
DELETE /api/boards/{id}                    — delete board
GET    /api/boards/{id}/articles           — list articles in a board
POST   /api/boards/{id}/articles           — add article to board
DELETE /api/boards/{id}/articles/{articleId} — remove from board
```

### New Migration

`V2__folders_and_boards.sql` — creates `folders`, `feed_folders`, `boards`, `board_articles` tables.

### SPA Serving

A `WebMvcConfigurer` that forwards non-API, non-static routes to `index.html` for client-side routing.

## Keyboard Shortcuts

| Key | Action | Context |
|---|---|---|
| `j` / `k` | Next / previous article | Article list |
| `n` / `p` | Next / previous feed | Feed panel |
| `Enter` | Open selected article in reading pane | Article list |
| `o` | Open original URL in new tab | Reading pane |
| `m` | Toggle read/unread | Article list / Reading pane |
| `s` | Toggle star | Article list / Reading pane |
| `b` | Add to board (opens board picker) | Reading pane |
| `v` | Open in Raindrop | Reading pane |
| `Shift+A` | Mark all as read in current feed | Article list |
| `/` | Focus search | Global |
| `Escape` | Close dialog / deselect / blur search | Global |
| `g` then `a` | Go to All Articles | Global (chord) |
| `g` then `s` | Go to Starred | Global (chord) |
| `g` then `b` | Go to Boards | Global (chord) |
| `?` | Show keyboard shortcut overlay | Global |
| `Tab` | Cycle panel focus | Global |

**Chord implementation:** `g` starts a chord — a small overlay shows "g → ..." until the second key is pressed or timeout (1 second).

**Auto-mark-as-read:** When a user navigates past an article with `j`, the previous article is automatically marked as read after a configurable delay (default 1 second). Can be disabled in settings.

## Article Rendering

- Article HTML content is sanitized with DOMPurify before rendering
- Consistent typography applied via CSS (line-height, font-size, heading styles, code block styling)
- Images rendered inline with max-width constraint
- "Open Original" button and `o` shortcut open the source URL in a new browser tab

## Integrations (v1)

- **Raindrop.io**: Full integration — save article to configured Raindrop collection via existing backend endpoint
- **Share menu**: Copy link, export as PDF (client-side), copy article text — no new backend work required
- **Integration settings**: Configure Raindrop API token and collection via Settings dialog, using existing `/api/integrations` endpoints

## Testing Strategy

### Frontend

| Layer | Tool | Scope |
|---|---|---|
| Unit | Vitest | Hooks, utilities, API client, keyboard chord logic |
| Component | Vitest + React Testing Library | Individual components: render, props, events |
| Integration | Vitest + MSW | Full panel interactions with mocked API |
| E2E | Playwright | Critical flows against real backend |

### Key Test Scenarios

- Feed panel: folder expand/collapse, unread counts update, feed selection
- Article list: filtering (unread/starred), sort, optimistic mark-read, scroll position
- Reading pane: HTML sanitization (malicious content stripped), open original, share menu
- Keyboard: all shortcuts, chord timeout, panel focus cycling
- Boards: create, add article, view, remove

### Backend (New Code)

- Repository tests: `@DataJdbcTest` + Testcontainers for `FolderRepository`, `BoardRepository`
- Controller tests: `@WebMvcTest` + `MockMvc` for folder and board endpoints
- Service tests: Mockito for folder/board business logic

## Out of Scope (v1)

- Mobile-responsive layout
- User authentication / multi-user
- Google Drive / Dropbox integrations
- Feed discovery / recommendations
- AI-powered features (summaries, categorization)
- Offline support / PWA
- Dark/light theme toggle (dark theme only for v1)
- Drag-and-drop reordering of feeds/folders
